# 料金シュミレーター追加プログラム

## プラン

- プランのボタンにアイコン画像を追加する
- アイコン画像にインストールが必要であればお願いします。
- アイコン、ボタンテキストの順で、並列表示。
- CSSは、docs/coding-rules/CSS_CODING_RULES.mdを参照。
- HTML、scriptは、PricingItem.tsx、EstimateDocument.tsxなどを参照。

1. コーディング
   - const codingIcon = "streamline-color:keyboard";
2. ホームページ制作
   - const homePageIcon = "twemoji:desktop-computer";
3. 特急案件
   - const urgentIcon = "twemoji:woman-running-light-skin-tone";

## 項目

- 以下のボタンについて、仕様変更
  - フェードインアウトアニメーション
  - スライドアニメーション
  - アコーディオンアニメーション
  - タブ切り替えアニメーション
  - その他のアニメーション

- ボタン選択後、アニメーションの数量が選択できるモーダルを追加でプログラムしてください。

- モーダル選択ボタン
  - 1
  - 2
  - 3
  - 4
  - 5
  - 6～10
  - 10～
  - 未定

- モーダルのデザインは、ModalDialog.tsx。データはpricing.tsxから。
- モーダルのタイトルは、「アニメーションの数量を選択してください」もしくは、もっと良い適切なタイトルがあれば改善してください。
- モーダルの数量単位は何が適切でしょうか？なければ、単位なしでも構いません。
- モーダル数量選択後、項目ボタンに選択した数量を記載してください。その際の表示は、項目ボタンテキストと数量を縦列にお願いします。（添付画像の下層ページボタンを参照してください。）
- モーダル数量選択後、項目ボタンに選択した数量の表記は、×2、×6～10などでお願いします。
- 各アニメーション単価に選択した数量を乗算して、金額を表示してください。
- 追加の仕様の流れとして、たとえば、スライドアニメーションボタンを選択して、 → 数量選択モーダル →「2」を選択 →「この結果で概算見積書を作成」→ 概算見積書の明細には品名スライドアニメーション・数量2と出力される・金額は￥10,000円と表示される → 小計、合計金額の表示も正しく表示される。
